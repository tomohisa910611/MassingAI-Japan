# MassingAI Japan：初心者向け起動手順

## 1. APIキーを設定する

プロジェクト直下の `.env.local` をメモ帳またはVisual Studio Codeで開きます。

```text
OPENAI_API_KEY=
```

`=` の右側に、控えてあるAPIキーを貼り付けて保存します。APIキーの前後に空白や引用符は付けません。

このファイルは自分のパソコンだけで使用します。画面、スクリーンショット、チャット、GitHubへ載せないでください。

## 2. アプリを起動する

プロジェクトフォルダーをVisual Studio Codeで開き、上部メニューの「ターミナル」から「新しいターミナル」を選びます。

最初の1回だけ、次を実行します。

```bash
npm install
```

続いて、アプリを起動します。

```bash
npm run dev
```

ブラウザで `http://localhost:3000` を開きます。終了するときはターミナルで `Ctrl` と `C` を同時に押します。

## 3. サンプル図面を試す

1. 「画像を選択」を押すか、点線の枠内へ求積図をドラッグします。
2. 選んだ画像のファイル名を確認します。
3. 「GPT-5.6 Solで解析する」を押します。
4. 解析後、細い青線と青い頂点で表示された敷地形状と寸法を確認します。
5. 修正する頂点をクリックし、前後2辺の長さを入力して候補位置を選びます。
6. 辺のダブルクリックで頂点追加、頂点のダブルクリックで削除を確認できます。
7. `Ctrl+Z` で元に戻し、`Ctrl+Y` でやり直せます。

APIクレジットを使用するのは、実際に「GPT-5.6 Solで解析する」を押したときです。

## 4. 今回の読取方針

- 四辺の長さだけから長方形を作りません。
- 元図面の境界線と頂点位置を優先します。
- 対角線、三角形の底辺・高さは、境界線ではなく形状確認に使用します。
- 座標表がある場合は、点名とX・Y座標を照合します。
- 座標から計算できる辺長は、画像から推測せず座標値を使用します。
- 同じ画像を再解析した場合は、初回に保存した同じ解析結果を使用します。

## 5. 困ったとき

- 「OPENAI_API_KEYが未設定」と表示されたら、`.env.local` を保存してからアプリを再起動します。
- APIキーのエラーが出たら、余分な空白がないか、APIクレジットが残っているか確認します。
- 画像が選べない場合はPNGまたはJPEGで、3MB以下か確認します。
- 読取が不正確な場合は、敷地境界と寸法が見やすい画像を使います。

APIキーが写った画面は共有しないでください。

## 6. 道路・真北の確認

解析後の「敷地の解析データ」に、図面から読み取った道路種別、路線名、道路幅員、真北角度が表示されます。
道路種別や幅員の印字が不明瞭な場合は、推測で確定せず「不明」と表示します。真北に数値角度がない場合は、矢印画像からの推定値として表示します。
